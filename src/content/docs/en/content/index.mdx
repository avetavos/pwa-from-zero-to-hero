In React you fetch data with `getServerSideProps`, `getStaticProps`, or `useEffect+fetch`. In Astro, data lives in the frontmatter `---` block — it runs at build time or on the server — or in content collections, which give you type-safe MD/MDX management. Zero browser-side fetching by default.

## Fetching data: React vs Astro

export const fetchReact = `// pages/blog.tsx (Next.js)
import { useEffect, useState } from 'react';

export default function Blog() {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    fetch('https://api.example.com/posts')
      .then(r => r.json())
      .then(setPosts);
  }, []);

  return (
    <ul>
      {posts.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}`;

export const fetchAstro = `---
// src/pages/blog.astro
const res = await fetch('https://api.example.com/posts');
const posts = await res.json();
---
<ul>
  {posts.map((post) => (
    <li>{post.title}</li>
  ))}
</ul>`;

<ReactAstro client:visible react={fetchReact} astro={fetchAstro} />

Notice: in Astro, `await fetch()` sits directly in the frontmatter. It runs at build time (static) or per-request (SSR). No `useEffect`, no loading state, no client-side JavaScript shipped for the fetch itself.

<Diff title="React data fetching vs Astro">
- React: three separate patterns — `getStaticProps` (build), `getServerSideProps` (per-request), `useEffect+fetch` (browser).
- Astro: **one pattern** — `await fetch()` in the frontmatter, no browser fetch shipped by default.
- React `useEffect+fetch` runs in the browser every page load; Astro frontmatter fetch runs **zero times** in the browser.
- Astro content collections add **type-safe MD/MDX management** with no third-party library required.
</Diff>
