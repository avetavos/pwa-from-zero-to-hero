In Next.js App Router, every `page.tsx` inside `app/` becomes a route. In Astro the same idea applies but the convention is simpler: **any `.astro` file inside `src/pages/` maps directly to a URL**. The filename is the path segment.

> Multi-page routing requires multiple files — open a full Astro project (`npm create astro@latest`) to run these examples locally.

## File → URL mapping

The table below shows how file paths inside `src/pages/` become URLs. This mirrors Next.js Pages Router conventions exactly — if you know `/pages`, you already know Astro routing.

| File path | URL |
|---|---|
| `src/pages/index.astro` | `/` |
| `src/pages/about.astro` | `/about` |
| `src/pages/blog/index.astro` | `/blog` |
| `src/pages/blog/first-post.astro` | `/blog/first-post` |
| `src/pages/contact.astro` | `/contact` |

## Side-by-side comparison

export const fileRoutingReact = `// Next.js App Router
app/
  page.tsx           → /
  about/
    page.tsx         → /about
  blog/
    page.tsx         → /blog
    first-post/
      page.tsx       → /blog/first-post

// Next.js Pages Router
pages/
  index.tsx          → /
  about.tsx          → /about
  blog/
    index.tsx        → /blog
    first-post.tsx   → /blog/first-post`;

export const fileRoutingAstro = `// Astro src/pages/
src/pages/
  index.astro        → /
  about.astro        → /about
  blog/
    index.astro      → /blog
    first-post.astro → /blog/first-post

// No page.tsx wrapper needed.
// The .astro file IS the page.`;

## What a page file looks like

A page `.astro` file is just a regular `.astro` component. The difference is location — files in `src/pages/` are treated as routes.

export const pageFileReact = `// app/about/page.tsx  (Next.js App Router)
export default function AboutPage() {
  return (
    <main>
      <h1>About</h1>
      <p>Welcome to the about page.</p>
    </main>
  );
}`;

export const pageFileAstro = `---
// src/pages/about.astro
import Layout from '../layouts/Layout.astro';
---
<Layout title="About">
  <main>
    <h1>About</h1>
    <p>Welcome to the about page.</p>
  </main>
</Layout>`;

<Diff title="Next.js pages vs Astro pages">
- Next.js App Router: every directory needs a `page.tsx` file to become a route.
- Next.js Pages Router: `pages/index.tsx` → `/`; mirrors Astro most closely.
- Astro: any `.astro` file in `src/pages/` becomes a route — no wrapping folder needed.
- Both: `index.astro` / `index.tsx` maps to the parent directory path (`/blog`, not `/blog/index`).
- Astro also supports `.md` and `.mdx` files in `src/pages/` as routes — useful for content sites.
</Diff>
