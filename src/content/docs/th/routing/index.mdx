React ให้อิสระในการเลือก router เอง ไม่ว่าจะเป็น react-router, TanStack Router หรือ convention ของ Next.js แต่ Astro ใช้แนวทางที่ต่างออกไป: routing **ถูกผูกอยู่กับ file system** วางไฟล์ `.astro` ไว้ใน `src/pages/` ก็กลายเป็น URL ทันที ไม่ต้องตั้งค่า ไม่ต้องใช้ `<Route>` และไม่ต้องครอบ provider

โมดูลนี้จะแมปทุก concept ด้าน routing และ layout ที่คุณรู้จักจาก react-router หรือ Next.js ให้ตรงกับสิ่งที่เทียบเท่าใน Astro

## บทเรียนในโมดูลนี้

| บทเรียน | สิ่งที่จะได้เรียนรู้ |
|---|---|
| [File-based routing](/astro-for-react-developers/th/routing/file-based-routing/) | `src/pages/*.astro` → URL segment |
| [Dynamic routes](/astro-for-react-developers/th/routing/dynamic-routes/) | `[slug].astro` + `getStaticPaths()` เทียบกับ `generateStaticParams` |

## โมเดล MPA

หากมาจาก React SPA หรือ Next.js อาจคาดว่าจะมี client-side router Astro ใช้โมเดล **Multi-Page Application (MPA)** เป็นค่าเริ่มต้น: ทุกการนำทางจะส่ง request ใหม่และ server ตอบกลับด้วย HTML ที่ render เสร็จแล้ว เหมือนกับที่เว็บทำงานก่อนยุค SPA — และทำให้ Astro เร็วมากในแบบ default

export const mpaReact = `// React SPA — client-side navigation
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}`;

export const mpaAstro = `<!-- Astro MPA — no router, no JS needed -->
<!-- src/pages/index.astro -->
---
import Layout from '../layouts/Layout.astro';
---
<Layout>
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
  <h1>Home</h1>
</Layout>

<!-- src/pages/about.astro  →  /about -->`;

<ReactAstro client:visible react={mpaReact} astro={mpaAstro} />

<Diff title="React SPA router vs Astro MPA routing">
- React SPA: routing เกิดขึ้น **ฝั่ง browser** — `BrowserRouter` สกัด click และสลับ component โดยไม่โหลดหน้าใหม่
- Astro MPA: routing เกิดขึ้น **ฝั่ง server** — แต่ละ URL แมปกับไฟล์ `.astro` ที่ render เป็น HTML; การนำทางคือ HTTP request ปกติ
- ทั้งคู่: รองรับ nested path, dynamic segment และ layout
- Astro เพิ่มเติม: เปิดใช้ `ViewTransitions` เพื่อ transition แบบ SPA ในขณะยังคงรูปแบบ MPA (ดูในบทเรียน Navigation)
</Diff>
